# BRIEF

- 目标: 新增 `/DEV_SDD:review_doc`，审查项目 `modules/**/SPEC.md` 是否覆盖 `docs/CONTEXT.md` 的模块功能定义。
- 输入: 目标项目根目录、`docs/CONTEXT.md`、`modules/**/SPEC.md`。
- 输出: `{status,message,data}`，包含每个模块的覆盖性、具体性、严谨性、可执行性评估与缺口。
- 规则: 默认使用当前激活项目；支持显式项目名或项目路径；只读，不改写项目文档。
- 覆盖判定: 逐模块比对 `职责/输入/输出/依赖`，缺失 SPEC 或关键字段未覆盖时给出 warning。
- 质量判定: SPEC 至少需要明确接口/类型或行为、错误或约束、测试或验收依据，才能视为足够具体/严谨/可执行。
- 验收1: 存在缺失 SPEC 或明显弱规格时，工具返回 `warning` 并列出模块问题。
- 验收2: 规格完整时，工具返回 `ok`，并说明模块级检查通过。
- 验收3: `--help` 含「用途」「示例」；`--json` 输出合法三字段结构。
